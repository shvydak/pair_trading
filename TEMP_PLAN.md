Детали незакрытых задач

4. Auto-close по z-score

- Backend: новый endpoint POST /api/triggers — сохраняет конфиг триггера (tp_zscore, sl_zscore, max_hours) к позиции
- Backend: фоновый asyncio.Task, который проверяет z-score для каждой активной пары каждые N секунд
- При срабатывании — вызывает тот же механизм что и ✕ M (market close через /api/trade action=close)
- Frontend: в таблице Strategy Positions — поля TP/SL z-score + max hours per row, кнопка "Set"
- Frontend: WebSocket уже стримит z-score — можно триггерить и на клиенте (проще, но ненадёжно)

5. Reconciliation (доделать)

- Сейчас: только 🗑 кнопка удаления DB-записи вручную
- Нужно: кнопка "Sync" — автоматически сравнивает DB с Binance, предлагает закрыть/удалить расходящиеся записи

6. Браузерные уведомления

- Notification API (разрешение при первом запуске)
- Триггеры: z-score пересёк entryZ/exitZ, смарт-ордер исполнен/отменён, PnL упал ниже порога

8. Kill switch

- Поле "дневной лимит убытка" в сайдбаре
- Backend считает суммарный PnL за сегодня из closed_trades + unrealized из enriched
- При превышении — блокирует кнопки Open в UI + опционально отправляет сигнал на бэкенд
